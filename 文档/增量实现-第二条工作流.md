上传素材后，不应该直接拿“原始视频文件”去检索知识库，而是要先把素材解析成 **素材画像 Material Profile**，再用素材画像去检索“适合它的剪辑技巧、结构模板和迁移规则”。

整体流程可以这样设计：

```text
用户上传素材
  ↓
素材解析与切分
  ↓
生成素材画像
  ↓
构造知识库检索 Query
  ↓
标签过滤 + 向量检索 + 规则匹配
  ↓
知识重排序
  ↓
返回可用剪辑技巧 / 结构模板 / 迁移策略
  ↓
生成剪辑方案
```

## 一、第一步：上传素材后先做素材解析

不要把一个完整视频当成一个整体处理，最好先切成多个片段。

### 1. 视频切分

可以按镜头切分：

```text
素材视频
├── shot_001：0s-2.4s
├── shot_002：2.4s-5.8s
├── shot_003：5.8s-9.1s
└── shot_004：9.1s-13.5s
```

每个 shot 都单独分析。

### 2. 提取视觉信息

对每个片段提取：

```text
场景：室内 / 户外 / 店铺 / 产品桌面 / 人物口播
主体：人 / 产品 / 食物 / 风景 / 工具
镜头类型：特写 / 中景 / 远景 / 俯拍 / 手持
动作：展示 / 使用 / 走路 / 对比 / 指向 / 拆箱
画面质量：清晰度 / 稳定性 / 亮度 / 构图
运动强度：静态 / 轻微运动 / 快速运动
情绪氛围：专业 / 温馨 / 兴奋 / 高级 / 搞笑
```

### 3. 提取音频信息

如果素材有声音，可以提取：

```text
是否有人声
是否有背景音乐
语速快慢
音量变化
音乐节奏 BPM
是否有明显卡点
情绪：轻快 / 紧张 / 治愈 / 热血
```

### 4. 提取文字信息

包括 OCR 和 ASR：

```text
画面文字 OCR
口播字幕 ASR
关键词
品牌名
价格
卖点
痛点词
行动引导词
```

## 二、第二步：生成“素材画像”

素材画像就是后续检索知识库的核心输入。

可以设计成这样：

```json
{
  "material_id": "mat_001",
  "video_type_candidate": ["产品种草", "教程", "测评"],
  "duration": 18.5,
  "global_tags": [
    "产品特写",
    "人物使用",
    "室内场景",
    "中快节奏",
    "卖点展示"
  ],
  "available_elements": {
    "has_product_closeup": true,
    "has_human_face": true,
    "has_before_after_comparison": false,
    "has_usage_process": true,
    "has_environment_shot": true,
    "has_speech": true,
    "has_text": false
  },
  "visual_style": {
    "brightness": "medium",
    "color_tone": "warm",
    "camera_motion": "slight",
    "composition_quality": "good"
  },
  "emotion": ["专业", "轻松"],
  "pace": "medium_fast",
  "shots": [
    {
      "shot_id": "shot_001",
      "time_range": "0-2.4s",
      "tags": ["产品特写", "静态展示", "清晰画面"],
      "suitable_usage": ["开头吸引", "卖点展示", "封面候选"]
    },
    {
      "shot_id": "shot_002",
      "time_range": "2.4-5.8s",
      "tags": ["人物使用", "操作过程", "中景"],
      "suitable_usage": ["功能演示", "教程步骤"]
    }
  ]
}
```

这个画像里最重要的是两类信息：

```text
1. 素材有什么？
例如：产品特写、人物使用、环境镜头、前后对比、口播、动作过程。

2. 素材适合做什么？
例如：适合做开头、适合做卖点展示、适合做教程步骤、适合做结尾转化。
```

## 三、第三步：根据素材画像构造检索 Query

你的系统可以自动把素材画像转成检索请求。

例如素材画像是：

```text
产品特写 + 人物使用 + 室内场景 + 中快节奏 + 有口播 + 无前后对比
```

系统可以生成检索 Query：

```json
{
  "task": "为上传素材检索合适的剪辑知识",
  "video_type": ["产品种草", "教程"],
  "material_tags": ["产品特写", "人物使用", "室内场景", "中快节奏", "有口播"],
  "available_elements": ["产品特写", "使用过程", "人物镜头"],
  "missing_elements": ["前后对比"],
  "target_goal": ["提高完播率", "强化卖点", "提高转化率"]
}
```

然后去知识库中检索：

```text
适合产品种草的结构模板
适合产品特写的开头技巧
适合人物使用镜头的卖点推进方式
适合中快节奏视频的剪辑技巧
不依赖前后对比素材的结尾策略
```

## 四、第四步：知识库检索不要只做向量检索，要做混合检索

建议采用：

> **标签过滤 + 向量检索 + 规则匹配 + 重排序**

而不是单纯 RAG。

### 1. 标签过滤

先用硬条件筛掉明显不适合的知识。

例如某条知识要求：

```json
"material_requirement": ["前后对比素材", "结果展示画面"]
```

但上传素材没有前后对比，那么这条知识就要降权，甚至过滤掉。

可以这样判断：

```text
素材有产品特写 → 可以召回“产品特写开头”
素材有人物使用 → 可以召回“使用场景演示”
素材没有对比画面 → 不优先召回“前后对比结尾”
素材有口播 → 可以召回“口播跳切 + 关键词字幕”
素材节奏较快 → 可以召回“快切 / 卡点 / 动态字幕”
```

---

### 2. 向量检索

把素材画像转成一句自然语言描述，再做 embedding 检索。

例如：

```text
上传素材包含产品特写、人物使用过程和室内展示画面，整体节奏中等偏快，有口播，适合制作产品种草或教程类短视频，需要突出卖点并提升完播率。
```

然后用这句话去检索知识库中的：

```text
description
applicable_scenarios
editing_actions
transfer_rule
example
```

这样可以找到语义上相关的技巧。

---

### 3. 规则匹配

规则匹配用来判断某条知识是否真的能落地。

例如知识库里有一条：

```json
{
  "name": "痛点开场",
  "material_requirement": ["痛点画面", "问题文本", "人物表情"],
  "editing_actions": [
    "前3秒展示痛点场景",
    "加入问题式字幕",
    "使用轻微放大或震动效果"
  ]
}
```

如果上传素材里没有明显痛点画面，但有产品结果画面，就可以触发 fallback：

```json
{
  "fallback_strategy": "若没有痛点画面，则使用结果画面前置，并配合疑问式字幕。"
}
```

这就是“迁移引擎”和普通检索系统的区别：
普通检索只告诉你“用痛点开场”；你的系统应该判断“当前素材能不能用痛点开场，如果不能，怎么替代”。

---

### 4. 重排序

最后给每条知识打分。

可以用一个简单公式：

```text
最终得分 =
0.35 × 语义相似度
+ 0.25 × 标签匹配度
+ 0.25 × 素材兼容度
+ 0.10 × 目标匹配度
+ 0.05 × 优先级
- 不适用惩罚
```

其中：

```text
语义相似度：素材描述和知识描述是否相近
标签匹配度：标签是否重合
素材兼容度：素材是否满足该技巧的使用条件
目标匹配度：是否符合提高完播率、点击率、转化率等目标
优先级：人工设定的经验权重
不适用惩罚：缺少关键素材时扣分
```

## 五、一个完整例子

假设用户上传了一组素材：

```text
素材内容：
- 有产品特写
- 有人物使用产品的过程
- 有室内环境
- 有口播
- 没有前后对比画面
- 没有明显剧情冲突
```

系统生成素材画像：

```json
{
  "video_type_candidate": ["产品种草", "教程"],
  "material_tags": ["产品特写", "人物使用", "口播", "室内场景", "中快节奏"],
  "missing_elements": ["前后对比", "剧情冲突"],
  "target_goal": ["强化卖点", "提高完播率"]
}
```

检索知识库后，召回结果可以是：

```text
Top 1：结果前置开头
原因：素材有清晰产品特写，适合先展示最终效果。

Top 2：口播跳切
原因：素材有口播，可以去掉停顿并提升节奏。

Top 3：关键词高亮字幕
原因：适合突出产品卖点、价格、功能词。

Top 4：使用过程三段式展示
原因：素材有人物使用过程，可以拆成“拿起—使用—结果感受”。

Top 5：无对比素材下的结尾转化
原因：素材缺少前后对比，因此不推荐强行做对比结尾，而是使用总结式推荐。
```

最后输出剪辑建议：

```text
0-3s：使用产品特写做结果前置开头，配大字标题。
3-8s：人物使用过程快切，突出核心功能。
8-13s：口播内容压缩，关键词字幕高亮。
13-17s：补充产品细节特写，加入轻微推近。
17-20s：总结卖点，用推荐式结尾。
```

## 六、检索模块可以这样写成系统流程

```text
Material Parser
素材解析器
负责切分镜头、识别主体、提取字幕、分析节奏。

Material Profiler
素材画像生成器
负责生成素材标签、场景标签、情绪标签和可用元素。

Query Planner
检索查询生成器
根据素材画像生成标签查询和语义查询。

Hybrid Retriever
混合检索器
同时进行标签检索、向量检索和规则匹配。

Knowledge Reranker
知识重排序器
根据素材兼容度和目标匹配度重新排序。

Editing Planner
剪辑方案生成器
将召回的知识转化为时间轴剪辑方案。
```

你可以画成架构图：

```text
上传素材
  ↓
[素材解析器]
  ↓
镜头切分 / OCR / ASR / 视觉标签 / 音频节奏
  ↓
[素材画像生成器]
  ↓
素材标签 + 可用元素 + 缺失元素 + 情绪节奏
  ↓
[Query Planner]
  ↓
标签 Query + 语义 Query + 规则条件
  ↓
[混合知识检索]
  ├── 标签过滤
  ├── 向量检索
  ├── 规则匹配
  └── 素材兼容性判断
  ↓
[知识重排序]
  ↓
剪辑技巧 / 结构模板 / 迁移规则
  ↓
[剪辑方案生成器]
  ↓
分镜脚本 / 时间轴 / 字幕包装 / 音乐卡点
```

## 七、知识库里的字段要支持“被检索”

为了让上传素材后能有效检索，每条知识最好有这些字段：

```json
{
  "id": "knowledge_001",
  "name": "口播跳切",
  "category": "剪辑技巧",
  "description": "通过删除口播中的停顿、重复和无效语气词，提高信息密度和观看节奏。",
  "applicable_scenarios": ["产品种草", "教程", "知识科普"],
  "required_materials": ["口播素材", "人物镜头"],
  "optional_materials": ["产品特写", "B-roll"],
  "avoid_conditions": ["情绪氛围需要慢节奏", "口播内容过短"],
  "matched_tags": ["口播", "快节奏", "提高完播率", "信息密度"],
  "editing_actions": [
    "删除停顿超过0.5秒的片段",
    "保留核心信息句",
    "在关键词处加入字幕高亮",
    "用B-roll遮盖剪切痕迹"
  ],
  "transfer_rule": {
    "source_pattern": "样例视频通过连续口播跳切保持高信息密度",
    "target_adaptation": "在新素材中识别口播段落，删除停顿和重复内容，并用产品特写遮盖剪辑点",
    "fallback": "若口播不足，则使用字幕旁白结构替代"
  },
  "priority": 4
}
```

这样上传素材后，系统才能判断：

```text
素材有口播吗？
有，则召回“口播跳切”。

素材有产品特写吗？
有，则可以配合 B-roll 遮盖剪切点。

素材是否适合快节奏？
适合，则提高该技巧排序。
```

## 八、如果还有“样例爆款视频”，检索要分两路

你的系统叫“爆款视频迁移引擎”，通常输入不只是新素材，还会有样例爆款视频。

这时检索应该分成两路：

```text
样例视频 → 提取爆款结构标签
新素材 → 提取素材可用标签
```

然后组合检索：

```text
样例标签：痛点开场 / 快切 / 关键词字幕 / 产品特写 / 强转化结尾
新素材标签：有产品特写 / 有人物使用 / 有口播 / 无前后对比
```

最终检索时问知识库：

```text
如何把“痛点开场”迁移到一组有产品特写、人物使用但没有痛点画面的新素材中？
如何把“快切节奏”迁移到中等运动强度素材中？
如何把“关键词字幕包装”迁移到有口播的素材中？
如果没有前后对比素材，如何替代“结果对比结尾”？
```

这样才能体现“迁移”，而不是简单套模板。

## 九、最推荐你的检索逻辑

可以概括成一句：

> **上传素材后，系统先生成素材画像，再基于“视频类型、素材可用元素、情绪节奏、目标效果、缺失条件”进行混合检索，从知识库中召回最适合当前素材的剪辑技巧、结构模板和迁移规则。**

具体实现上，建议你采用：

```text
第一层：标签过滤
先筛出视频类型、情绪、素材条件匹配的知识。

第二层：向量召回
根据素材画像的自然语言描述召回语义相关知识。

第三层：规则校验
判断该知识是否真的能被当前素材执行。

第四层：LLM 重排序
让大模型根据“是否适合当前素材”和“是否符合爆款迁移目标”重新排序。

第五层：方案生成
把 Top-K 知识组合成可执行剪辑时间轴。
```

最终你的系统不是在问：

> “知识库里有什么剪辑技巧？”

而是在问：

> **“基于当前上传素材，哪些爆款剪辑技巧能用？怎么用？如果素材不满足条件，怎么替代？”**
