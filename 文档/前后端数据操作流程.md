# 前后端数据操作完整流程

## 用户使用链路全景

### 1. 左侧边栏：视频资源管理区
- 用户上传**样例视频** → 标记 `is_reference=True`
- 用户上传**素材视频** → 标记 `is_reference=False`
- 支持对任意视频新增、删除操作
- 点击任意视频 → 中间播放器自动切换播放该视频

**数据流向：**
```
前端 FormData POST /api/orchestration/jobs
  → 后端写入 Redis 共享记忆 inputs.uploaded_videos
  → 立刻持久化落盘，服务重启不丢失
```

---

### 2. 中间工作区：播放器 + 时间线
- 视频播放器支持：播放 / 暂停 / 快进 / 快退 / 音量调节
- 播放器下方可视化多轨道时间线
  - 每个片段拖拽移动、调整时长、修改属性
  - 新增片段、删除片段、重新排序
  - 修改操作实时写入 Redis，SSE 广播同步
- 三个 Tab 区域：
  - **样例视频** Tab：展示上传的样例视频 + 解析结果卡片
  - **素材视频** Tab：展示已索引的AIGC生成素材
  - **结果视频** Tab：展示最终渲染导出的成片 + 完整剪辑思路

**数据流向：**
```
前端 PATCH /jobs/{job_id}/timeline/segments/{segment_id}
  → 修改 Redis 中 edit_timeline 结构化数据
  → 所有连接该 job_id 的 SSE 客户端实时收到更新通知
  → 多用户协作场景画面秒级同步
```

---

### 3. 右侧边栏：Agent 实时对话区
- 用户在底部输入框发送自然语言指令
- 「流式竖思考链」实时渲染 Agent 运行全过程：
  - 每个 Agent 开始执行 → 立即收到 `step_start` 事件
  - Agent 中间输出思考内容 → 增量追加展示到思考区
  - Agent 执行完成 → 收到 `step_write` 事件，弹出结果卡片
- 结果卡片支持点击展开，弹出 DetailModal 查看完整报告
- 多轮对话场景：用户发送新指令 re-submit → 不需要刷新页面，SSE 长连接持续复用，新一轮 Agent 事件直接接在后面

**数据流向：**
```
用户输入指令 POST /jobs/{job_id}/re-submit
  → 后台 Agent 逐个串行执行
  → 每一步调用 append_event() 写入 Redis 事件流 List
  → SSE 长连接 0.5秒内 推送到前端
  → 前端 useSSEEventSource 收到事件，增量渲染到对话区
```

## 全链路数据存储在 Redis 高速缓存
所有状态 100% 存在 Redis，随取随用：

| Redis Key | 数据结构 | 存放内容 | TTL |
|----------|---------|---------|-----|
| `job:{job_id}:shared_memory` | JSON | 完整 SessionSharedMemory 共享记忆对象 | 24h |

## 前端完整对接架构

### 1. useSSEEventSource Hook
文件路径：`frontend/src/hooks/useSSEEventSource.ts`

核心能力：
- 自动管理 EventSource 生命周期
- 指数退避重连策略 1s → 2s → 4s → 8s
- `last_event_id` 断点续传，重连不丢事件
- SSE 连接失败自动降级到 1.5s 轮询兜底
- 组件卸载自动关闭连接释放资源

### 2. 接入 AgentConversationPanel
```tsx
import { useSSEEventSource, type SSEEventPayload } from '../hooks/useSSEEventSource'

const [currentJobId, setCurrentJobId] = useState<string | null>(null)
const [messages, setMessages] = useState<Message[]>([])
const [agentNodes, setAgentNodes] = useState<AgentNode[]>(initialMockNodes)

useSSEEventSource({
  jobId: currentJobId,
  onEvent: (evt: SSEEventPayload) => {
    if (evt.event_type === 'step_start') {
      // 更新对应 Agent 节点状态为 running
      setAgentNodes(prev => prev.map(n =>
        n.name === evt.agent_name ? { ...n, status: 'running' } : n
      ))
    }
    if (evt.event_type === 'step_write') {
      // 追加 Agent 结果消息到对话区
      setMessages(prev => [...prev, 转换成AgentMessage(evt)])
      // 对应节点状态置为 done
      setAgentNodes(prev => prev.map(n =>
        n.name === evt.agent_name ? { ...n, status: 'done' } : n
      ))
    }
    if (evt.event_type === 'step_fail') {
      setAgentNodes(prev => prev.map(n =>
        n.name === evt.agent_name ? { ...n, status: 'error' } : n
      ))
    }
  }
})
```
