建议你用 **“主 Prompt + 分文件 Prompt + 校验 Prompt”** 三段式，让大模型模拟生成一个小型知识库。

核心原则是：**让大模型生成的知识库结构，尽量对齐你已有 `ReferenceAnalyzerAgent` 的输出字段**，尤其是 `structural_slots / transition_style_template / caption_style_template / transfer_rules`，这样后续无论模板来自“样例视频解析”还是“知识库生成”，都能被同一套 SlotMatcher / EditPlanner 消费 。

# 1. 总控 Prompt：让大模型知道它在生成什么

你可以先给大模型这个系统 Prompt：

```text
你是一个短视频风格知识库生成器，目标是为一个“爆款视频迁移剪辑系统”生成小型结构化知识库。

系统背景：
用户可能没有上传样例视频，因此系统需要根据用户输入的风格需求，从知识库中检索风格模板，并生成可执行的剪辑结构。
你的任务不是写普通的剪辑教程，而是生成可以被 Agent 系统直接消费的 JSON 知识库。

核心要求：
1. 所有知识必须结构化，不能只写自然语言描述。
2. 每个风格模板必须能生成 structural_slots。
3. 每个 structural_slot 都要包含：
   - role
   - duration
   - creative_function
   - information_function
   - required_visual_type
   - required_motion
   - shot_size
   - caption_requirement
   - audio_sync
   - transition_out
   - importance
4. 风格模板要能指导素材匹配，而不是只描述风格感觉。
5. 输出内容必须适合后续用于：
   - 风格检索
   - 素材片段匹配
   - 剪辑规划
   - 字幕生成
   - 转场选择
   - 音效选择
6. 不要生成无法执行的抽象词，比如“高级”“丝滑”“好看”必须落到具体规则上。
7. 输出必须是合法 JSON，不要包含 Markdown，不要解释。

你需要生成一个小型知识库，包含以下文件内容：
1. style_templates
2. editing_techniques
3. caption_formulas
4. matching_rules
5. asset_segment_schema
```

# 2. 风格模板生成 Prompt

这是最重要的 Prompt。建议你**一个风格一个风格生成**，不要一次性让它生成 5 个，质量会更稳。

## Prompt 模板

```text
请为短视频自动剪辑系统生成一个风格模板 JSON。

风格名称：{style_name}
风格类型：{type_label}
目标时长：{target_duration} 秒
目标平台：抖音 / 小红书
画幅：9:16

要求：
1. 输出一个完整 JSON 对象。
2. 该 JSON 必须能作为知识库中的 style_template 使用。
3. 必须包含 5 个 structural_slots：
   - hook
   - context
   - highlight_01
   - highlight_02
   - ending
4. 每个 slot 必须是可匹配、可执行、可剪辑的。
5. 每个 slot 必须描述需要什么类型的素材，而不是描述最终效果。
6. transition_out 必须使用具体可执行类型：
   cut / whip_pan / zoom_cut / blur / flash_black / flash_white / dissolve / fade_out
7. audio_sync.beat_position 不能写 any，必须从以下选择：
   first_strong_beat / bar_change / transition_downbeat / phrase_release / weak_beat
8. 字幕要求必须明确是否需要字幕、字幕位置、字幕语义功能。
9. 必须包含 fallback_strategy，用于素材不满足时的降级方案。
10. 输出必须是合法 JSON，不要 Markdown，不要解释。

JSON schema 如下：

{

  "structural_slots": [
    {
      "slot_id": "",

      "creative_function": "",
      "information_function": "",
      "required_visual_type": [],
      "required_motion": [],

        "need_caption": true,

        "position": "",
        "semantic_role": "",
        "caption_formula_hint": ""
      },
      "audio_sync": {
        "beat_position": "",
        "sfx": ""
      },
      "transition_out": "",
      "importance": "",
      "match_priority": [],
      "copy_risk": ""
    }
  ],
  "caption_style_template": {
    "subtitle_density": "",
```

# 8. 质量检查 Prompt

大模型生成完之后，建议再让另一个 Prompt 做检查。

```text
你是短视频知识库质量检查器。
请检查下面的 JSON 是否适合作为自动剪辑系统的知识库。

检查标准：
1. JSON 是否合法。
2. 字段是否完整。
3. 是否能直接用于 structural_slot 和素材 segment 匹配。
4. 是否存在过于抽象、不可执行的描述。
5. transition_out 是否都是可执行类型。
6. audio_sync.beat_position 是否具体。
7. required_visual_type 是否描述素材需求，而不是最终效果。
8. fallback_strategy 是否能处理素材不足的情况。
9. embedding_text 是否适合向量检索。
10. 是否有中英文风格不统一的问题。

请输出：
{
  "valid": true/false,
  "problems": [],
  "suggested_fixes": [],
  "revised_json": {}
}

待检查 JSON：
{把生成的 JSON 粘贴到这里}
```

# 9. 推荐生成顺序

你可以按这个顺序让大模型生成：

```text
1. 先生成 5 个 style_template
2. 再生成 editing_techniques.json
3. 再生成 caption_formulas.json
4. 再生成 matching_rules.json
5. 再生成 asset_segment_schema.json
6. 最后逐个用质量检查 Prompt 校验
```

不要一次性生成全部。建议一轮只生成一个文件。

# 11. 最关键的 Prompt 设计原则

你要反复约束大模型：

```text
不要生成教程，要生成可执行知识。
不要只描述风格，要生成 structural_slots。
不要只说“高级感”，要说明需要什么素材、什么镜头运动、什么转场、什么字幕、什么卡点。
不要让后续 Agent 猜，要把匹配条件写清楚。
```

也就是让大模型生成的知识库从一开始就服务于这条链路：

```text
用户风格需求
→ 检索 style_template
→ 得到 structural_slots
→ 解析用户素材 segments
→ slot 与 segment 匹配
→ 生成剪辑草稿
```

这样你的小知识库虽然不大，但已经可以支撑完整 demo。
