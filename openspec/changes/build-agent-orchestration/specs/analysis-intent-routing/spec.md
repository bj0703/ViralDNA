## MODIFIED Requirements

### Requirement: 系统将自然语言解析为多 agent 执行计划
系统 SHALL 将用户自然语言请求解析为一个多 agent 执行计划，说明需要调用哪些 agent 及其执行顺序，而不仅识别单一分析维度。

#### Scenario: 识别完整迁移创作意图
- **WHEN** 用户输入"帮我把这个样例的结构套到我的素材上剪一版"
- **THEN** 系统返回一个包含样例分析、素材分析、槽位匹配、缺口补齐和剪辑编排等 agent 的执行计划

#### Scenario: 识别单维度分析意图
- **WHEN** 用户输入"只看这个视频的节奏结构"
- **THEN** 系统返回仅包含对应分析步骤的执行计划，而不调用下游迁移 agent

### Requirement: 系统结合会话记忆状态规划前置条件
系统 SHALL 在生成执行计划时读取当前会话共享记忆状态，以判断所需 agent 的前置产物是否已就绪，并据此补全或调整计划。

#### Scenario: 缺少前置样例分析
- **WHEN** 用户请求结构迁移，但共享记忆中尚无 `reference_analysis`
- **THEN** 系统在计划中前置补上样例分析步骤，或返回需要先完成样例分析的说明

#### Scenario: 缺少用户素材
- **WHEN** 用户请求剪辑编排，但共享记忆中尚无 `asset_index`
- **THEN** 系统返回"需要先上传素材"的澄清，而不规划无法执行的下游步骤

### Requirement: 系统输出标准化意图规划结果
系统 SHALL 以标准化结构返回意图规划结果，其中保留兼容字段 `intent` 与 `analysis_scope`，并新增表示 agent 步骤序列的 `plan` 字段及置信度。

#### Scenario: 规划结果明确
- **WHEN** 系统能稳定将用户请求映射为受控 agent 集合内的计划
- **THEN** 返回包含 `intent`、`analysis_scope`、`plan` 和高置信度的标准化结构

#### Scenario: 规划结果不明确
- **WHEN** 用户请求无法明确映射为受控 agent 计划
- **THEN** 系统返回默认回退计划或请求用户澄清，而不规划超出受控集合的动作

### Requirement: 系统限制在受控 agent 集合内规划
系统 SHALL 将执行计划限制在一组受控 agent 集合内（样例分析、素材分析、槽位匹配、缺口补齐、剪辑编排及现有分析动作），不规划超出该集合的任意动作。

#### Scenario: 请求在受控集合内
- **WHEN** 用户请求可由受控 agent 集合满足的迁移创作动作
- **THEN** 系统将其规划为对应 agent 步骤序列

#### Scenario: 请求超出受控集合
- **WHEN** 用户请求当前系统尚不支持的能力
- **THEN** 系统返回"不支持当前动作"或建议可替代的 agent 计划
