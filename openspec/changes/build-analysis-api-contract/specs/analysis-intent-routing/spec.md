## ADDED Requirements

### Requirement: 系统提供自然语言意图识别接口
系统 SHALL 提供自然语言意图识别接口，用于将用户输入解析为受控的分析动作。

#### Scenario: 识别全维度视频分析
- **WHEN** 用户输入“解析这个视频”或语义等价指令
- **THEN** 系统返回 `video-analysis` 或等价的全维度分析意图

#### Scenario: 识别特定分析维度
- **WHEN** 用户输入“只分析节奏”或“识别这个视频的 hook”
- **THEN** 系统返回对应的受控分析意图和作用范围

### Requirement: 系统输出标准化意图结果
系统 SHALL 以标准化结构返回意图识别结果，其中至少包含 `intent`、`analysis_scope`、`target_video_id` 和 `fallback_behavior`。

#### Scenario: 识别结果明确
- **WHEN** 系统能够稳定识别用户意图
- **THEN** 返回标准化意图结构并标记高置信度

#### Scenario: 识别结果不明确
- **WHEN** 用户指令无法明确映射到单一分析动作
- **THEN** 系统返回默认回退策略，例如执行全维度分析或请求用户缩小范围

### Requirement: 系统限制在受控意图集合内路由
系统 SHALL 将第一版自然语言分析请求限制在一组受控分析意图集合内，而不是无边界执行任意动作。

#### Scenario: 请求在受控集合中
- **WHEN** 用户请求脚本结构分析、节奏分析、高光识别或导出结构化结果
- **THEN** 系统将其路由到对应的分析动作

#### Scenario: 请求超出受控集合
- **WHEN** 用户请求当前模块尚不支持的复杂编辑或生成行为
- **THEN** 系统返回“不支持当前动作”或建议可替代分析动作
