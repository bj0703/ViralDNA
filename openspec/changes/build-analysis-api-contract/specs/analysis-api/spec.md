## ADDED Requirements

### Requirement: 系统使用固定的豆包请求配置
系统 SHALL 在当前版本中仅使用 `ARK_API_KEY`、`ARK_ENDPOINT_ID` 和 `ARK_BASE_URL` 这三个配置项调用豆包模型，并 SHALL 通过 HTTP `chat/completions` 请求发起调用。

#### Scenario: 配置齐全时发起请求
- **WHEN** 后端具备 `ARK_API_KEY`、`ARK_ENDPOINT_ID` 和 `ARK_BASE_URL`
- **THEN** 系统使用这三个配置构造豆包请求并发起模型调用

#### Scenario: 配置缺失时拒绝分析
- **WHEN** 任一必要配置缺失
- **THEN** 系统返回明确的配置错误，而不是使用未声明的默认扩展参数继续请求

### Requirement: 系统提供分析任务创建接口
系统 SHALL 提供分析任务创建接口，用于接收一个已选定视频和分析范围信息，并创建一个可追踪的分析任务。

#### Scenario: 创建视频分析任务
- **WHEN** 前端提交选中视频标识、分析范围和可选指令
- **THEN** 系统返回唯一 `job_id`、任务状态和基础会话信息

#### Scenario: 请求缺少必要参数
- **WHEN** 前端未提供目标视频标识或无效分析范围
- **THEN** 系统返回明确的参数错误响应，而不是创建任务

### Requirement: 系统提供分析任务状态查询接口
系统 SHALL 提供任务状态查询接口，并 SHALL 返回任务的当前状态、更新时间和可用结果概览。

#### Scenario: 查询进行中的任务
- **WHEN** 前端请求一个处于处理中状态的任务
- **THEN** 系统返回 `queued` 或 `processing` 状态以及最近更新时间

#### Scenario: 查询已完成任务
- **WHEN** 前端请求一个已完成的分析任务
- **THEN** 系统返回 `completed` 状态和可获取结果的引用信息

### Requirement: 系统提供标准化分析结果接口
系统 SHALL 提供机器可读的标准化分析结果接口，用于返回分析模块产出的完整结构化数据。

#### Scenario: 请求完整分析结果
- **WHEN** 前端或下游模块请求已完成任务的分析结果
- **THEN** 系统返回包含基础元数据、脚本结构、节奏结构、置信度和可用性状态的标准化结果

#### Scenario: 请求未完成任务的结果
- **WHEN** 前端请求一个尚未完成任务的完整结果
- **THEN** 系统返回未就绪状态，而不是伪造结果内容

### Requirement: 系统不直接透传模型原始文本
系统 SHALL 对豆包模型的原始响应进行标准化解析后再返回给前端，而不是直接透传模型原始文本。

#### Scenario: 模型返回正常文本内容
- **WHEN** 豆包接口返回可解析的模型内容
- **THEN** 系统将其转换为标准化分析结果或展示模型

#### Scenario: 模型返回内容不符合预期
- **WHEN** 豆包接口返回内容缺失或结构异常
- **THEN** 系统返回标准化错误或降级结果，而不是将不可控原始响应直接暴露给前端

### Requirement: 系统提供前端展示模型接口
系统 SHALL 提供面向前端工作台的展示模型接口，用于聚合播放器、时间轴和结构面板所需字段。

#### Scenario: 前端请求展示模型
- **WHEN** 前端请求已完成任务的展示模型
- **THEN** 系统返回适合渲染时间轴片段、右侧结构卡片和片段详情浮层的聚合数据结构

#### Scenario: 展示模型包含片段联动信息
- **WHEN** 系统返回展示模型
- **THEN** 结果中包含每个时间片段的时间范围、标签、摘要、置信度和联动标识

### Requirement: 系统定义统一任务状态
系统 SHALL 为分析相关任务使用统一状态集合，包括 `queued`、`processing`、`completed` 和 `failed`。

#### Scenario: 任务成功完成
- **WHEN** 分析流程正常结束
- **THEN** 系统将任务状态标记为 `completed`

#### Scenario: 任务处理失败
- **WHEN** 分析流程出现不可恢复错误
- **THEN** 系统将任务状态标记为 `failed` 并返回可读错误信息
