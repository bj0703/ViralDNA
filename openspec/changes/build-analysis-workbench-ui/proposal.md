## Why

现有样例分析能力已经可以返回结构化结果，但缺少一个适合真实创作与演示的前端工作台。为了让用户能够边看视频、边发指令、边理解分析结果，需要一个围绕“播放器 + 时间轴 + 结构面板”的分析工作台，而不是单一聊天框或调试页。

## What Changes

- 增加分析工作台前端界面，采用左中右三栏布局。
- 左栏用于视频资源管理和当前视频选择。
- 中栏用于视频播放、片段查看和时间轴可视化。
- 右栏用于自然语言输入、分析快捷操作和结构化结果展示。
- 支持时间轴、播放器和右侧结构结果联动，提升分析结果的可解释性与可操作性。

## Capabilities

### New Capabilities
- `analysis-workbench-ui`: 提供面向视频分析任务的三栏工作台界面、时间轴可视化和结构面板联动能力。

## Impact

- 影响未来 `frontend` 项目的页面结构、组件层级、状态管理和交互设计。
- 需要消费 `analysis-api` 与 `analysis-intent-routing` 的结果，形成完整前后端分离闭环。
- 直接影响评审演示观感，是系统“可解释展示能力”的核心入口。
