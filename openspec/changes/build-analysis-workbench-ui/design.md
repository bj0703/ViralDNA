## Context

视频分析模块如果只返回 JSON 或简单文本，很难支撑评审演示，也不利于后续结构迁移使用。对于这个课题，更合适的形态是“分析工作台”：左边选视频，中间看视频与时间轴，右边输入自然语言并查看结构结果。

你当前的三栏想法是正确的，但需要进一步收敛职责，避免右侧只剩聊天、时间轴只剩静态线条。工作台应该强调“联动”和“结构化展示”。

## Goals / Non-Goals

**Goals:**
- 定义三栏工作台的布局职责和核心交互。
- 让播放器、时间轴和结构面板形成联动。
- 支持用户选中视频后，通过自然语言触发分析。
- 让分析结果同时具备“可读性”和“可调试性”。

**Non-Goals:**
- 不在本次 change 中定义最终视觉品牌规范。
- 不在本次 change 中扩展结果视频生成或复杂编辑器能力。
- 不在本次 change 中支持多人协作、版本对比或云端草稿系统。

## Decisions

### 1. 保留三栏布局，但明确每栏职责
- 左栏：资源区，负责样例视频、素材视频、结果视频和草稿箱入口
- 中栏：主工作区，负责播放器和时间轴
- 右栏：分析区，负责自然语言输入、快捷动作和结构结果面板

原因：
- 贴合视频创作工具的使用习惯。
- 有利于把“资源管理”“主视觉区域”“智能分析”分开。

### 2. 右栏采用“输入区 + 结果面板”双层结构
右栏不能只显示聊天记录，而应包含：
- 自然语言输入框
- 快捷分析按钮
- 结构化结果 tabs

原因：
- 分析结果不能淹没在聊天流里。
- 用户需要快速切换“概览 / 脚本 / 节奏 / 风险 / 原始结果”。

### 3. 时间轴采用“片段块 + 悬浮详情”方式
时间轴不只展示单点，而要以区间块形式展示结构片段，并支持 hover 气泡和点击跳转。

原因：
- 视频结构天然是时间区间，不是离散节点。
- 更适合展示 hook、中段、CTA 和节奏分段。

### 4. 以联动为第一优先级
工作台的核心不只是“展示结果”，而是：
- 点击右侧结构卡片，高亮时间轴片段
- 点击时间轴片段，播放器跳转到对应时间
- 播放器时间变化时，同步高亮当前片段

原因：
- 这是分析结果真正可用的关键。
- 能显著增强可解释性和评审展示效果。

## Risks / Trade-offs

- [时间轴组件复杂度较高] -> 第一版先做分段渲染、hover 和点击跳转，缩放与拖拽可延后。
- [右栏信息过载] -> 用 tabs 和折叠卡片控制密度。
- [三栏同时可调宽度实现成本较高] -> 第一版先支持基本拖拽分栏，后续再优化动画和自适应细节。

## Migration Plan

前端工作台优先以消费标准分析 API 为前提进行开发。若当前后端仅提供原始结果接口，可以先用中间映射层适配，待 `analysis-api` change 实现后再切换到正式契约。

## Open Questions

- 第一版是否需要支持移动端，还是先聚焦桌面端演示？
- 时间轴是否需要同时支持“脚本结构视图”和“节奏结构视图”的切换？
- 右栏是否需要保留完整对话历史，还是只保留最近一次分析上下文？
