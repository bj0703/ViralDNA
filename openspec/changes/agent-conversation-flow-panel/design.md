## Context

emo_transfer 是一个多 Agent 编排系统，包含 ReferenceAnalyzer、AssetIndexer、SlotMatcher、EditPlanner、GapResolver、FinalVideoRenderer 等多个 Agent 协作执行流水线。用户不仅需要看到最终输出，还需要清晰感知整个执行流程的透明度。参考 TRAE 的对话交互风格和 AI 研究助手的节点流转进度条，我们现在在右栏操作区打造一套完整的 Agent 流式对话面板，完全复用之前定义的三栏布局和 Soft Lavender 紫蓝色彩主题。

## Goals / Non-Goals

**Goals:**
- 定义用户消息 + Agent 回复的完整消息格式，Agent 回复天然拆分为可折叠思考区 + 最终结果区两层结构。
- 实现垂直流式逐字输出模式，支持 Agent 思考过程边生成边展示。
- 实现圆点节点流转进度条，可视化标记多 Agent 流水线的当前执行位置和已完成状态。
- 实现格式化结果卡片系统，每个 Agent 完成任务后输出结构化卡片，支持多卡片横向翻页浏览。

**Non-Goals:**
- 不在本次设计中实现完整 WebSocket 长连接通信，第一版先用模拟流式数据驱动视图。
- 不实现复杂的消息编辑、重发、历史消息搜索功能，这些后续扩展。
- 不实现 Markdown 全语法渲染，第一版聚焦核心文本、代码块、结构化卡片渲染。

## Decisions

### 1. 消息体数据结构定义

采用 User / Agent 两种消息类型的统一消息队列：

```typescript
type Message =
  | {
      type: 'user';
      id: string;
      content: string;
      timestamp: number;
    }
  | {
      type: 'agent';
      id: string;
      agentName: string;
      agentIcon?: string;
      thinking: {
        isExpanded: boolean;
        content: string; // 支持流式增量追加
      };
      result: {
        content: string;
        cardType: 'overview' | 'script' | 'timeline' | 'assets' | 'report';
      };
      status: 'running' | 'done' | 'error';
      timestamp: number;
    };
```

决定理由：
- 清晰区分两层结构，思考过程和最终结果字段完全分离，避免逻辑混乱。
- 支持增量追加流式内容，适配 SSE / WebSocket 逐字输出场景。

### 2. 对话消息气泡布局

```
┌─────────────────────────────────────────┐
│                                         │
│              用户消息（右对齐）           │
│  ┌───────────────────────────────┐      │
│  │ 我要分析这个视频的情感节奏       │      │
│  └───────────────────────────────┘      │
│                                         │
│ Agent 头像  Agent名称                    │
│  ▾ 思考过程（可折叠展开/收起）            │
│     这里是逐层展开的 Thought 流式内容     │
│     ... ...                             │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │  📋 参考视频分析结果卡片           │  │
│  │                                   │  │
│  │  这里是格式化最终结果展示          │  │
│  └───────────────────────────────────┘  │
│                                         │
└─────────────────────────────────────────┘
```

- 用户消息气泡：右对齐，背景色 --color-primary-500 紫蓝主色，白色文字。
- Agent 消息气泡：左对齐，背景透明，Agent 头像 + 名称在左上角。
- 「思考过程」区块：浅灰背景，有折叠箭头按钮，默认收起。点击箭头展开全部思考内容。

### 3. 圆点节点流转进度条设计

```
  ◉ ReferenceAnalyzer ──── ◉ AssetIndexer ──── ○ SlotMatcher ──── ○ EditPlanner
  (已完成)                   (运行中)             (待运行)          (待运行)
```

位置：固定在右栏对话面板的顶部，横向全宽展示。

状态定义：
| 状态 | 圆点样式 | 含义 |
|------|----------|------|
| 已完成 | 实心黑圆 ✅ | Agent 执行完毕，展示黑底白色对勾 |
| 运行中 | 紫蓝实心圆 ⚡ | Agent 当前正在执行，有呼吸动画效果 |
| 待运行 | 空心白圆 ○ | Agent 还未启动执行 |

决定理由：
- 完全参考截图中的研究计划/信息检索/数据分析/撰写报告的节点流转风格，用户一眼就能看懂流水线进度。
- 用圆点 + 连接线的极简视觉语言，信息密度高且不占太多垂直空间。

替代方案考虑：
- 方案 A：垂直时间轴 → 占垂直空间太多，挤占下面对话内容的位置。
- 方案 B：文字标签进度条 → 视觉辨识度远不如圆点节点清晰。

### 4. 缩略封面卡片 + 模态详情弹窗

在对话面板的底部放置「结果缩略卡片横向滚动区」，每个小卡片只放精简的封面信息，不展示任何内部详情内容：

```
┌──────────────────────────────────────────────────────────────────────┐
│  ◀  生成结果 (3/7)  ▶                                              │
│  ┌───────────┐  ┌───────────┐  ┌───────────┐                    │
│  │ 📋 概览   │  │ 📝 脚本   │  │ 🎬 资产   │ → 横向滑动          │
│  │ 缩略封面  │  │ 缩略封面  │  │ 缩略封面  │                    │
│  └───────────┘  └───────────┘  └───────────┘                    │
└──────────────────────────────────────────────────────────────────────┘
```

**点击缩略封面卡片 → 弹出居中模态大窗口，全屏展示该 Agent 的完整结构化详情内容。**

决定理由：
- 小缩略卡片区域空间极度节省，完全不拥挤，右栏 400px+ 宽度下横排 4 个都很宽松。
- 详情模态弹窗可以利用整个屏幕的宽度，把大段文本、长列表、时间线图表完整展示出来，体验更好。
- 避免了小卡片里塞不下大量结构化内容的拥挤问题。

缩略封面卡片内容规则：
- 大图标 emoji / 小图标 + 卡片类型名称（2个字）
- Agent 名称小字 + 生成时间戳小字
- 卡片背景：淡紫蓝渐变浅色背景
- 点击弹出半透明黑色蒙层 + 居中 Modal 大窗口

### 5. 布局整体分层结构

右栏垂直方向从上到下：
1.  顶部：Agent 节点流转进度条（固定高度 60px）
2.  中间：对话消息流滚动区（自适应填充剩余空间）
3.  底部：用户自然语言输入框区（固定高度 ~72px）

## Risks / Trade-offs

- [流式输出过程中页面滚动定位不准] → Mitigation：每次新消息追加时自动滚动到底部，用户手动向上滚动后暂停自动滚动。
- [节点流转数量超过 5 个时横向空间不够] → Mitigation：节点超出屏幕宽度时允许横向滚动进度条，保证所有节点都能看到。
- [思考过程内容过长占满屏幕] → Mitigation：默认全部收起，用户按需点击展开，避免内容溢出。

## Migration Plan

1.  定义 Message 类型系统和模拟数据。
2.  实现圆点节点流转进度条组件。
3.  实现单条用户消息 + Agent 消息组件。
4.  实现思考过程折叠/展开交互。
5.  实现格式化结果卡片和横向翻页组件。
6.  接入模拟流式数据，整体跑通全流程演示。
