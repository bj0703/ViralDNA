## Context

左栏现在只是简单占位内容，需要升级为三层垂直堆叠的视频资源管理区，给用户提供直观的样例视频上传、素材视频批量上传、剪辑多版本结果查看能力。完全复用已有的 Soft Lavender 紫蓝色彩主题，不改动外层 WorkbenchLayout 布局架构。

## Goals / Non-Goals

**Goals:**
- 左栏垂直排列三个独立可折叠板块，从上到下是样例视频区、素材视频区、剪辑结果多版本区。
- 样例视频区：单视频占位 + 上传按钮 + 缩略图 + 删除按钮。
- 素材视频区：支持批量上传多个，网格布局展示缩略图，每个卡片带删除。
- 剪辑结果多版本区：纵向时序列表展示，每个版本带版本号、时间戳、缩略图和操作按钮。

**Non-Goals:**
- 不在本次实现中接入真实后端文件上传接口，第一版先用本地模拟状态驱动 UI。
- 不实现复杂的视频预览播放器，只用封面缩略图展示。
- 不实现拖拽排序功能，第一版保持顺序固定。

## Decisions

### 1. 可折叠板块通用头部

每个板块都有统一的可折叠头部设计：
```
▼  板块标题  (上传按钮 + 计数标签)
--------------------------------
   内容区域
```
- 头部左侧：展开/收起箭头 + 板块名称文字
- 头部右侧：小的「+ 上传」按钮 + 右上角小圆角计数徽章（显示该板块的视频数量）
- 点击箭头即可切换展开/收起状态，状态持久化到 localStorage

### 2. 样例视频上传区

这是最顶部的第一个板块，用于用户上传参考样例视频：
- 空状态：显示一个虚线上传占位框，提示「点击上传参考样例视频」
- 已上传状态：显示视频封面缩略图（圆角卡片，100% 宽度），右下角悬浮一个小的删除 X 按钮
- 限制：只允许上传 1 个样例视频，是单选模式

### 3. 素材视频上传区

中间第二个板块，用于用户批量上传多个素材视频：
- 空状态：显示虚线上传占位
- 已上传状态：2 列网格布局，每个小卡片 1:1 正方形缩略图，右下角悬浮删除按钮
- 支持上传多个，卡片数量不限，自动横向排满换行

### 4. 剪辑结果多版本保存区

最底部第三个板块，纵向时序列表展示：
```
V1.0  2026-06-06 15:30  [预览] [删除]
V1.1  2026-06-06 16:10  [预览] [删除]
V2.0  2026-06-06 17:05  [预览] [删除]
```
- 每个版本项从左到右：版本号标签 + 生成时间戳 + 预览按钮 + 删除按钮
- 纵向排列，新生成的版本永远排在最上面

## Risks / Trade-offs

- [左栏宽度最小只有 360px，2 列网格素材视频卡片不能太挤] → Mitigation：每个卡片设为 宽高比 1:1 140px，在 420px 左右的宽度下刚好放下两个。
- [三个板块都展开可能内容溢出] → Mitigation：整个左栏本身是可滚动容器，自动开启垂直滚动条。
- [用户刷新页面丢失上传状态] → Mitigation：所有板块的上传状态临时存在 localStorage 模拟持久化。

## Migration Plan

1. 定义视频资源通用 TypeScript 类型。
2. 实现可折叠板块通用头部组件。
3. 实现样例视频上传板块。
4. 实现素材视频批量上传板块 2 列网格。
5. 实现剪辑结果多版本时序列表。
6. 整体拼装替换原来的 LeftSidebarContent 占位组件。
