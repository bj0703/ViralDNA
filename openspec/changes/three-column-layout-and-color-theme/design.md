## Context

emo_transfer 是一个面向视频创作的 AI 情感转译系统。我们需要从零构建前端工作台的基础，不再沿用之前旧设计的假设，独立定义一套清晰的三栏分区布局系统和浅色优先的清新紫蓝主题。参考 NexusAI 现代 SaaS AI 产品的视觉风格，这是整个前端架构的第一层地基，决定了后续所有页面组件的容器约束和视觉语言。

## Goals / Non-Goals

**Goals:**
- 定义三栏分区的精确布局结构、默认尺寸、可拖拽边界和最小最大宽度约束。
- 建立浅色模式优先的完整色彩系统，采用清新紫蓝主色调，包含中性色、紫蓝主色、语义色三级色板。
- 明确各栏的职责边界，避免未来出现职责混淆。

**Non-Goals:**
- 不在本次设计中实现具体业务组件（播放器、时间轴、分析面板），只做布局容器和基础主题层。
- 不做复杂的响应式多端适配，第一版聚焦桌面端 1920x1080+ 场景。
- 不定义动效、动画和过渡曲线，这些后续单独扩展。

## Decisions

### 1. 三栏分区定义

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  顶部导航栏（高度 52px，全站固定）                                           │
├───────────┬───────────────────────────────────────┬─────────────────────────┤
│           │                                       │                         │
│   左栏     │              中栏                      │         右栏            │
│  (资源区)  │             (主工作区)                  │       (操作区)         │
│           │                                       │                         │
│ 默认 240px │           自适应填充                   │     默认 320px         │
│  最小 180px│           最小 500px                  │     最小 260px         │
│  最大 400px│                                       │     最大 500px         │
└───────────┴───────────────────────────────────────┴─────────────────────────┘
```

- **左栏（资源区）**：垂直排列视频库、草稿箱、素材库导航。左侧固定宽度区，主要放导航树和缩略图列表。
- **中栏（主工作区）**：绝对中心，自适应填充剩余空间，承载视频播放器、时间轴画布等核心视觉元素。这是用户注意力焦点所在，必须保证最大可用面积。
- **右栏（操作区）**：放置自然语言输入、分析参数、结果详情面板。从右侧展开的操作流区域。

决定理由：
- 参考了 Figma、CapCut 专业创作工具的经典三栏分配范式。
- 中栏自适应填充保证了视频创作场景下的最大化画布。
- 左右两栏都支持拖拽调整，给不同屏幕尺寸的用户灵活空间。

### 2. 分栏拖拽机制

- 两个可拖拽的分隔条，宽度 4px，hover 时扩展为 8px 并显示拖动指示器。
- 左-中分隔条可拖动范围：180px 到 400px。
- 中-右分隔条可拖动范围：自适应计算，保证中栏不小于 500px，右栏不小于 260px。
- 拖动过程中实时更新宽度，松开后保存到本地 localStorage，下次进入恢复上次布局。

替代方案考虑：
- 方案 A：支持三栏同时拖拽但不设最小宽度约束 → 风险是中栏可能被挤爆，播放器区域过小无法使用。
- 方案 B：每栏提供折叠按钮点击隐藏 → 第一版先做基础拖拽，折叠功能后续加。

### 3. 浅色优先清新紫蓝主题

参考 NexusAI 现代 SaaS AI 产品风格，采用 **Soft Lavender（柔熏衣紫）** 主题定位，整体明亮通透，浅灰白底色搭配柔和紫蓝主色，给人干净、现代、充满 AI 感的视觉体验。

| Neutral 200 | #E5E7EB | 分割线、边框 |

| Neutral 800 | #1F2937 | 标题、大字号强文本 |
| Neutral 900 | #111827 | 最深黑色，logo 和主标题 |

| 语义 | HEX 值 | 用途 |

| Info | #06B6D4 | 信息提示、进行中状态 |

#### 功能区底色约定
- 页面全局背景：`#F5F5F5` (Neutral 50)
- 左栏背景：`#FFFFFF` (Neutral 100)
- 中栏背景：`#F5F5F5` (Neutral 50)
- 右栏背景：`#FFFFFF` (Neutral 100)
- 卡片悬浮层：`#FFFFFF` + 16px 圆角 + 极浅阴影 `0 4px 20px rgba(0, 0, 0, 0.05)`

决定理由：
- 浅色模式符合当前主流 AI SaaS 产品的审美趋势，显得现代、干净、通透。
- 柔和紫蓝色（#6366F1）比常规蓝色更有个性但又不跳脱，有明显的 AI 产品识别度。
- 高对比度的黑白文本系统保证了极佳的可读性。

替代方案考虑：
- 方案 A：深色模式优先 → 虽然视频看更沉浸，但整体产品定位是 AI 分析工作台，浅色更适合阅读和长时间办公。
- 方案 B：纯蓝色主色调 → 过于传统，缺少 AI 产品的独特识别感。

### 4. 间距与圆角基础规范

- 间距单位：4px 基础单元，8/12/16/20/24/32/48 递进。
- 卡片圆角：12px
- 按钮圆角：8px
- 输入框圆角：8px
- 组件内部边距：16px

## Risks / Trade-offs

- [三栏拖拽交互在小屏笔记本上可能拥挤] → Mitigation：设置最小宽度硬约束，屏幕小于 1366px 时自动触发简易布局提示，不强制挤爆。
- [浅色模式下文本对比度必须保证可访问性] → Mitigation：所有文本颜色经过 WCAG AA 对比度校验，正文文本对比度 ≥ 4.5:1。
- [硬编码 HEX 色值后续不好改] → Mitigation：所有颜色通过 CSS Variables / CSS-in-JS token 统一管理，不直接在业务代码中写死色值。

## Migration Plan

1. 先创建前端项目骨架，定义 CSS 全局变量。
2. 实现顶部导航栏 + 三栏容器布局。
3. 接入分栏拖拽交互并保存布局状态。
4. 验证所有色板对比度和视觉一致性。
